Hybrid设计

发表于 2019-03-07 | 分类于 iOS

前言

写前言就跟高中时候写作文非要写个题记一样，成了习惯。有时候就觉的自己神经质，不过想起来高中时候有几次考试的作文是全班少有的高分，还是很激动的。

现在较为成熟稳定的商业App，基本都是由Native和H5混合搭建的，H5页面开发效率高，灵活性强，无需版本依赖。听起来优势很多，甚至有App整个模块都是H5页面开发的，因此也会带来用户体验的问题，我见过许多App，虽然是一整个模块H5，但是页面跳转的时候，在当前页面显示进度条真是叫人受不了。
上面只是基于用户层面讲，而对于开发层面来说，混合开发需要考虑许多问题，便有了此文章。
一般需要解决的问题：
1.数据交互
2.功能定制
…
数据交互是Native-Web双向的，部分功能的定制也是通过数据的交互实现的，定制化的功能又可以通过插件化的思想去实现，使得HybridView更加轻便。

Hybrid设计架构图

为了调用的安全，以及对Native--Web的交互统一,所以WebView并没有暴露给调用者，这也符合开放封闭的设计原则，对外可扩展，但不可修改。
在设计HybridView的时候，先后有几个版本，在HybridEngine之前WebView本是HybridView私有属性，但随着定制化的功能越来越多，如：需要分享功能、需要主题功能，一种比较好的解耦方式，就是对功能进行插件化的管理，所以，又引入了HybridWebView管理WebView，HybridPlugin作为WebView插件的基类，HybridEngine是HybridView处理Webview的操作者，同时也是外部插件整合到WebView上的处理者，其它关于Native-Web是如何方法调用传递参数交互的，请移步百度、谷歌。

Custom是一些定制化的UI样式，视具体业务而定，如左上角的返回按钮、弹出菜单等等。
